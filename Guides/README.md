# Guides

Standalone walkthroughs and setup guides for the cohort, gathered here so they live in one place. Each guide is its own repository (or file). Read whichever one you need, in any order, though if you are setting up from scratch the first three below make a natural sequence.

## Getting set up (a natural order)

**1. [upitt-anaconda-setup-guide](https://github.com/DatJavaClass/upitt-anaconda-setup-guide)**
Start here if you have nothing installed yet. A step-by-step walkthrough for installing Anaconda, creating the course conda environment (`upitt`, Python 3.10.20, with the class packages), and running your first Jupyter notebook. Covers **Windows, Linux, and macOS**, using a plain "Type / Press / You should see" format so you always know what to expect next.

**2. [CourseraToVSCode](https://github.com/DatJavaClass/CourseraToVSCode)** — *"Zero to VS Hero"*
Once Anaconda and the `upitt` environment exist, this guide adds VS Code on top: installing it, learning what extensions are and installing the four you need, and pointing VS Code at the `upitt` environment you already made so you can run your notebooks there. Nothing on the Python side changes, you are just adding a friendlier place to work.

**3. [Goodbye AI - On the Road with VS Code](Goodbye%20AI%20-%20On%20the%20Road%20with%20VS%20Code.pdf)** (PDF)
The follow-up to the Coursera-to-VS-Code guide. Five short stops to a clean VS Code with no AI looking over your shoulder: build a fresh, empty **No-AI** profile, switch off VS Code's built-in AI features and grey "ghost text" suggestions, load the two extensions you need, connect a notebook to your `upitt` kernel, and finish with a one-click desktop launcher for the whole thing. Your normal VS Code is left exactly as it was.

## Bonus tutorial

**[UCFCECS_ALL_PYTHON_LOOPS_INTRO](https://github.com/DatJavaClass/UCFCECS_ALL_PYTHON_LOOPS_INTRO)**
A short, standalone loops tutorial (originally a tutoring example for the UCF Assisted Learning Center). It uses nested loops to pull the letters of `python` out of a scrambled string, in order, and walks through the code line by line. A gentle way to get comfortable with nested `for` loops and the idea of tracking a position as you scan.
